# 完善扫码配对确认界面

## Goal

让配置页在生成配对二维码后，立即在同一个界面提供移动端验证码输入与确认入口，用户无需再切换到待处理配对列表即可完成配对授权。

## Requirements

- 点击生成二维码后，在二维码展示区域同时显示：二维码、移动端验证码输入框，以及确认/批准按钮。
- 输入框绑定本次生成二维码返回的 `pairingId`，接收移动端扫码后显示的六位数字验证码。
- 输入内容仅允许数字，最多六位；未满六位时确认按钮不可用。
- 点击确认后调用现有配对批准接口，提交本次 `pairingId` 与验证码。
- 批准成功后保留现有行为：刷新/更新设备列表，清理二维码和验证码状态，并关闭或结束当前二维码流程。
- 保留现有待处理配对列表及其操作，避免影响其他配对流程。
- 不修改后端 API 契约；新增或调整用户可见文案时同步维护中英文翻译。

## Acceptance Criteria

- [x] 生成二维码后，同一弹窗同时显示二维码、验证码输入框和确认按钮。
- [x] 输入框只接受最多六位数字，输入不足六位时确认按钮禁用。
- [x] 输入六位验证码并确认后，向对应 pairing ID 发起批准请求，成功后二维码和验证码界面被清理，设备列表更新。
- [x] 现有待处理配对列表流程仍可正常使用。
- [x] 相关前端测试覆盖新增交互，且 `pnpm lint`、`pnpm typecheck`、`pnpm test`、`pnpm build` 通过。

## Notes

- 复用现有 `approvalCodes`、`onApprovalCodeChange` 与 `onApprove` 状态和回调，避免重复实现配对批准逻辑。
- 复用现有验证码输入、确认按钮及翻译文案，只有确有需要时才新增中英文文案。
