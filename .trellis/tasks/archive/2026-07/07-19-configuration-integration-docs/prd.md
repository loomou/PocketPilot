# 编写配置页前后端对接文档并核对流程

## Goal

基于仓库当前后端实现，为 Local Admin 配置页编写可供前后端开发和联调使用的独立中文、英文对接文档，并逐项核对现有配置页实现与后端真实流程是否一致。

## Requirements

- 以后端路由、服务、Zod schema、状态存储和测试为事实来源，不根据界面或历史说明臆测接口行为。
- 覆盖 Local Admin 页面实际使用的能力：启动与访问边界、CSRF、状态、运行配置、任务配置、目录浏览/检查/授权、配对二维码与批准、设备撤销、审计记录、维护边界及错误处理。
- 分别生成完整中文文档和完整英文文档，不在同一文件中混排双语段落。
- 文档应明确 HTTP 方法、路径、请求头、请求体、响应体、状态变化、关键校验、错误处理和推荐调用顺序。
- 文档应区分浏览器临时状态、服务端持久状态、仅下次启动生效的配置，以及安全敏感或仅限终端的操作。
- 对照 `apps/local-admin` 当前实现，逐项判断其请求、状态更新和交互流程是否与后端一致。
- 发现不一致时，在文档中明确记录严重度、影响、证据路径和建议修正方向；除非核对所必需，本任务不直接修改产品流程代码。
- 在 README 中加入中英文 Local Admin 对接文档入口，保持现有文档链接风格。

## Acceptance Criteria

- [x] `docs/` 下存在独立的英文与简体中文 Local Admin 对接文档。
- [x] 两份文档都覆盖所有配置页使用的后端接口、核心数据契约、推荐流程和错误/安全约束。
- [x] 两份文档内容语义一致，但分别使用英文和简体中文完整表达。
- [x] 文档包含现有前端与后端流程核对结论，以及每项不一致的代码证据与建议。
- [x] README 提供两份文档的入口。
- [x] 文档中的路径、字段和流程均经过后端代码与测试交叉验证。
- [x] 文档格式检查、项目 lint/typecheck/test/build 或适用的质量检查通过。

## Notes

- 预期文件名遵循现有移动端指南风格：`local-admin-integration-guide.en.md` 与 `local-admin-integration-guide.zh-CN.md`。
- 这是文档与流程审查任务，不新增后端 API。
