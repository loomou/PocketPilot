# Implementation plan: Local admin workspace authorization

## Preconditions

- Do not edit production code until the user approves this plan and `task.py start` changes the task status to `in_progress`.
- After start, load `trellis-before-dev` for the frontend and backend specs before the first production edit.
- Keep `tasks.workspaceRoots` as the sole persistent authorization source and preserve the explicit Save/Discard workflow.

## Ordered implementation

1. **Establish shared path-policy primitives**
   - Add/adapt `src/tasks/workspace-path-policy.ts`.
   - Implement canonical directory resolution, canonical identity keys, component-boundary containment, volume/UNC-root detection, exact duplicate checks, saved-root identity inspection, and nested coverage metadata.
   - Add focused cross-platform tests, including Windows case/UNC/8.3-safe fixtures and symlink/junction identity behavior where the platform supports it.

2. **Add the revisioned authorization coordinator**
   - Introduce the shared coordinator/service around `SettingsRepository`.
   - Implement current-settings reads, available-root inspection, mobile discovery, candidate admission, and validated task-settings replacement.
   - Implement revision retry plus short commit/admission serialization exactly as defined in `design.md`.
   - Test unavailable preservation/recovery, new unavailable rejection, canonical duplicate rejection, nested-root retention, high-risk confirmation, and deferred-canonicalization save/admission races.

3. **Wire one coordinator into runtime owners**
   - Construct one coordinator in `src/runtime/agent-runtime.ts`.
   - Inject it into `TaskManager` and local-admin app/configuration routes.
   - Route task-settings reads/writes and capacity reads through the shared owner without adding a second store.
   - Preserve existing listener lifecycle and runtime-settings behavior.

4. **Build and expose the local directory browser**
   - Add an injected filesystem/platform directory-browser service.
   - Implement virtual roots, Windows drives, POSIX root/home, absolute direct navigation, parent navigation, directories-only filtering, deterministic sorting, bounds, and truncation metadata.
   - Register `/admin/directories/browse` and `/admin/directories/inspect` only on the local app with Zod schemas and safe errors.
   - Add app-injection tests for valid browsing/inspection, files filtered out, invalid/relative paths, inaccessible paths, malformed payloads, CSRF/origin rejection, bounded listings, and remote-listener 404 isolation.

5. **Strengthen task-settings save validation**
   - Change `PUT /admin/configuration/tasks` to call the coordinator.
   - Add write-only canonical high-risk confirmation paths to the request contract while keeping the response/persisted settings shape unchanged.
   - Preserve unchanged unavailable saved rows, reject new unavailable/non-directory roots, persist canonical available roots, reject exact duplicates, and keep nested roots/order.
   - Add regression tests proving an unchanged unavailable row does not block unrelated configuration saves.

6. **Apply authorization to discovery and runtime admissions**
   - Make `authorizedWorkspaceRoots()` availability-aware and update `GET /v1/workspaces` to await it.
   - Reuse coordinator admission in all workspace/session catalog and creation paths.
   - Add missing rechecks to `activateSdkSession`, `resumeTask`, and every `submitSdkMessage` admission.
   - Ensure an already executing turn is not interrupted or terminalized by root removal; keep interrupt/close/current-approval resolution available.
   - Add TaskManager and route tests for removal, reauthorization, unavailable recovery, nested explicit roots, currently executing turn completion, denied next message/resume/activation, and retained history.

7. **Update remote contract documentation**
   - Change route/OpenAPI descriptions to “currently available configured authorized roots.”
   - Regenerate the mobile OpenAPI artifact through the normal build path.
   - Verify deterministic generated bytes and the unchanged response shape.

8. **Extend the local-admin browser API client**
   - Add Zod schemas/types/functions for browse and inspect responses in `apps/local-admin/src/api/local-admin.ts`.
   - Extend task-settings save input with high-risk confirmations without leaking transport shapes into components.
   - Add API decoder/error tests for valid, malformed, and non-2xx responses.

9. **Implement Configuration tabs and authorization table**
   - Refactor `configuration-view.tsx` into General and Authorized directories tabs while retaining one form/save bar.
   - Remove the newline workspace-roots textarea.
   - Add summary, empty state, status/coverage/high-risk badges, Add, and Remove actions.
   - Keep additions/removals in `configurationDraft.tasks.workspaceRoots`; maintain inspection and high-risk confirmation as transient state.
   - Ensure Discard and successful Save reconcile transient metadata with the saved snapshot.

10. **Implement the accessible directory-browser flow**
    - Add a feature-local modal/dialog shell with focus restore, Escape/cancel, accessible names, and busy/disabled states.
    - Add root/home shortcuts, breadcrumbs/up, absolute address input, directory rows, truncation notice, and Authorize action.
    - Add the second high-risk confirmation step for filesystem/volume roots.
    - Test cancellation/no-change, navigation, direct paths, duplicate feedback, high-risk confirmation, add/remove draft behavior, and modal keyboard/accessibility behavior.

11. **Complete bilingual resources and state-preservation coverage**
    - Add all new `zh-CN` and `en` messages through the locale registry/resource types.
    - Keep path/directory data opaque and notices semantic.
    - Test locale switching while the authorization tab, unsaved rows, address input, modal, and loaded listing are active.

12. **Run full quality review and update Trellis contracts**
    - Load and follow `trellis-check` after implementation.
    - Review cross-layer data flow: UI draft → local API → coordinator commit → discovery/runtime admission.
    - Update relevant frontend/backend Trellis specs for the new directory APIs, canonical/unavailable semantics, revocation admission points, and race contract.
    - Inspect `git diff --check`, generated OpenAPI changes, and final repository status before requesting commit approval.

## Review gates and rollback points

- **Gate A — path policy:** do not connect UI or routes until canonical identity, nested-root, unavailable-root, and cross-platform tests pass.
- **Gate B — concurrency:** do not rely on revocation semantics until the deterministic save/admission race test proves the post-save guarantee.
- **Gate C — backend boundary:** prove local-only route isolation and CSRF/origin behavior before frontend integration.
- **Gate D — runtime behavior:** prove current-turn completion plus next-admission denial before updating user-facing copy.
- **Gate E — frontend state:** prove Save/Discard and locale-switch preservation before full build.
- No database rollback is needed. If the feature is reverted, the existing string-array setting remains readable by the old code.
